Feature Status Overview

Live User Authentication
Email + password signup/login. Profile creation on first sign-in. Pending profile restore for email-confirmation flow on web.
Live Zone Check-In
GPS-based zone entry. Users select social mode (Open / Selective / Quiet) and mood. Sessions end on checkout or app close. PostGIS geofencing.
Live People at This Venue
Live list of checked-in users at the same zone. Shows name, interests, age range, kickoff prompt. Respects new check-in privacy setting.
Live We Met
Mutual connection request between checked-in users. Pending / Confirmed states. Direct message thread unlocked on both-sides confirm.
Live Direct Messages
Thread-based DMs between We Met pairs. Real-time via Supabase subscriptions. Message thread at /messages/[wemetId].
Live Zone Pulse
Social post feed inside a zone. Posts visible to everyone in the zone right now. Hidden posts filtered at query level (admin moderation).
Live Zone Chat
Real-time group chat inside a zone. Hidden messages filtered at query level. Reported messages flow to admin panel.
Live Universal Feed
Discovery feed showing all zone_posts across the app. Includes venue subscription sections: MY VENUES chip row + FROM YOUR VENUES feed cards.
Live Afterglow
Session recap shown after checkout. Venue visited, duration, connections made, mood summary.
Live Badges
Achievement badges awarded for first check-in, streak milestones, connections made. Displayed on profile.
Live Push Notifications
We Met requests, message pings, venue promotion alerts. Notifications tab in bottom nav. Supabase push token registry.
Live Venue Dashboard
Venue owners see live check-in count, age breakdown, interest cloud, subscriber count. Quick actions to manage all venue features.
New Venue Subscriptions (Follow)
Users tap "+ Follow" on any venue page. Creates a subscription row. Venue's promos and announcements appear in their feed. Unfollow from My Venues.
New Venue Promotions
Venue owners create promotions (title, discount label, description, feed toggle). Shown on venue page and in followers' feeds. Owner can delete anytime.
New Venue Announcements
280-char venue announcements pushed to followers. Optional universal feed post. History shown to venue owner. Venue owners can delete past announcements.
New My Venues
Profile nav item listing all venues the user follows. Tap to visit. Unfollow with confirmation. Empty state guides new users to find venues.
New Check-in Privacy
Toggle in Edit Profile: Full (name + interests + age + kickoff shown) vs Minimal (name only). Saved to checkin_visibility on profiles table.
Live Venue Highlights
Venue owners pin up to 5 highlight items (photo/text) shown at top of the zone page. Reorderable. Admin-controlled.
Live Admin Panel
Admin-only screens: user list, venue approval queue, content reports. Actions use server-side RPC with is_admin double-check.
Live Content Moderation
Admins can hide pulse posts and chat messages. Hidden content filtered at query level — never reaches the client.
Live PWA / Web
Expo web build with service worker registration. Full feature parity with native. Safe area insets applied across all screens.
Pending Auto-subscribe on Check-In
Jacob's idea: prompt users to follow a venue automatically when they check in. Currently manual. Planned follow-up.
Pending active_sessions RPC Privacy
The RPC that returns checked-in users needs updating to null-out interest_tags, kickoffs, age_range when checkin_visibility = 'minimal'. Backend SQL task.
Roadmap Instagram / TikTok Social Auth
Let users connect social handles to their profile. Show follower count as social proof to others at the venue.

Venue Owner — Full Journey

1
Apply as a venue
On the signup screen, owner selects "I own or manage a venue." They fill out the venue application form at /zone/create:
  • Venue name and description
  • Venue type (bar, restaurant, gym, coffee shop, etc.)
  • Radius (geofence size in meters — how far out check-in triggers)
  • GPS location auto-detected or manually confirmed
Application submitted → profile.venue_status = 'pending', profile.is_venue_owner = true.
2
Waiting for approval
The venue owner sees an "Under Review" screen when they open Venue Dashboard. They cannot access dashboard tools until approved. The pending card shows:
  • "Your venue application is being reviewed — usually 24–48 hours"
  • A contact email: support@herenow.app
  • Back to Home button
Profile nav shows "Venue Status" with an amber ⏱ Pending pill badge.
3
Admin reviews and approves
Admin opens /admin/venues. Sees all pending applications with venue name, owner name, and type. Approves or denies. On approval:
  • profiles.venue_status → 'approved'
  • A zone row is created (or linked if already attached)
  • Venue owner receives a push notification (planned)
Admin access: Jacob and Jamie are granted is_admin = true via the SQL migration. Any profile with this flag can access all admin routes.
4
Venue dashboard is live
Once approved, the owner sees the full dashboard at /venue/dashboard:
  • Live check-in counter — pulsing dot when anyone is in
  • Subscriber count — how many users are following the venue
  • Age range breakdown — anonymous bar chart (when check-ins active)
  • Top interests in the room — tag cloud (when check-ins active)
Quick action grid: Edit Venue · Add Event · Highlights · Promotions · Announce.
5
Market to followers
Promotions (/venue/promotions):
  • Title (required, max 80 chars)
  • Discount label — "30% off drinks", "Free entry before 10pm" (max 50 chars)
  • Details / description (max 400 chars)
  • Toggle: post to universal feed (all HereNow users) vs followers only
Announcements (/venue/announcements):
  • Up to 280 chars — like a tweet, direct to followers
  • Toggle: post to universal feed
  • History shown, delete anytime
Both appear in followers' FROM YOUR VENUES feed section immediately.
6
Manage venue appearance
Highlights (/venue/highlights): Pin up to 5 items shown at the top of the zone page — featured menu items, specials, upcoming events, photos.

Edit Venue (/venue/edit): Update name, description, type, geofence radius, and GPS location at any time.

Add Events (/zone/event/create): Create timed events (trivia night, DJ, happy hour) that show in the zone's event feed.

Regular User — Full Journey

1
Sign up
Screen at /(auth)/signup:
  • Email + password
  • Display name (what others see at venues)
  • Username (unique handle)
  • "I own or manage a venue" checkbox — routes to venue application if checked
Email confirmation required before first sign-in. On web, profile created automatically after email confirm via pending-profile localStorage restore.
Login fix shipped: Pressing Enter on the password field now submits the login form — no more having to manually tap "Sign In."
2
Set up profile
Profile edit at /profile/edit:
  • Display name, bio (160 chars), avatar photo (web upload)
  • Age range — shown to others at venues (or hidden via privacy)
  • Interests — up to 16 tags (Music, Gaming, Art, etc.)
  • Conversation kickoff — one-liner shown on your People card
  • Check-in Privacy toggle (new): Full (all visible) vs Minimal (name only)
3
Explore the home feed
Home tab (/(tabs)/index): Zone map showing nearby venues and events. Tap a zone chip to open it.

Feed tab (/(tabs)/feed):
  • MY VENUES — horizontal scroll chips for venues you follow. Tap to jump to the zone.
  • FROM YOUR VENUES — promotions and announcements from followed venues, sorted newest first.
  • DISCOVERY — universal post feed from all zones.
4
Arrive at a venue — zone page
Zone page at /zone/[id]:
  • Venue name, type, active event (if any)
  • + Follow button (new): Subscribe to get the venue's future promos in your feed. Bell icon 🔔 when following.
  • Venue highlights pinned at top
  • People tab — who's checked in right now
  • Pulse tab — zone social feed
  • Chat tab — live group chat
  • Events tab — upcoming events
  • Check In button — opens check-in screen
5
Check in
Check-in screen at /check-in/[zoneId]:
  • Choose Social Mode: Open (visible to all) / Selective (visible to interests-match) / Quiet (hidden from People list)
  • Choose Mood: Open to chat / Selective / Just vibing
  • Tap "Check In" → session row created, appears in People list for others
Privacy: if user has set Minimal check-in privacy, their interests, age, and kickoff are hidden from other users' People cards — only their name shows.
Planned: Auto-subscribe prompt on first check-in — "Follow this venue to see their deals?" (one-tap follow at check-in moment).
6
Connect with people
From the People tab:
  • Tap a person's card to view their full profile (respects their privacy setting)
  • Tap "We Met" to send a connection request
  • Other person sees a pending notification; accepts or declines
  • Both accept → DM thread unlocked at /messages/[wemetId]
Post to Pulse — share a moment to the zone's social feed (text or image).
Chat in real-time with everyone in the zone via the Chat tab.
7
Check out & Afterglow
Check out via the zone page or the active session card on profile. Session ends, user exits People list. Afterglow screen shows:
  • Venue visited, time spent
  • Connections made this session
  • Any badges earned
8
Manage venues you follow
Profile → My Venues (/my-venues):
  • List of all followed venues with follow date and type
  • Tap venue to jump to its zone page
  • Unfollow with confirmation — stops promos appearing in feed
  • Empty state explains how to follow from a venue page

New Database Tables (this PR)

Table Purpose Key columns

Bug Fixes & Code Quality (this PR)

Fix Details
Enter key on login Password TextInput now has returnKeyType="go" + onSubmitEditing={handleLogin}. Pressing Enter/Go submits the form.
Safe area insets — 4 screens check-in/[zoneId], we-met, zone/create, zone/event/create all had hardcoded paddingTop: 56. Now use useSafeAreaInsets() — correct on notch phones and iPads.
KeyboardAvoidingView — 2 screens zone/create and zone/event/create now wrap in KAV so keyboard doesn't cover text inputs on iOS.
.single() → .maybeSingle() afterglow/[sessionId] and check-in/[zoneId] — changed to .maybeSingle() so missing rows return null instead of PGRST116 error.
Hidden content moderation fetchPulse and fetchChat now filter .eq('is_hidden', false). Admin-hidden posts/messages never reach clients.
Messages list bottom padding app/messages/index.tsx list style now has paddingBottom: 80 so the last thread isn't hidden behind the tab bar.
Redundant auth calls venue/promotions and venue/announcements were calling getUser() inside the INSERT object on every save. Moved to top of handler, single call with early exit guard.
